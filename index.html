Network Engineering Implementation

RelayDoodle uses a pure Java Socket implementation (Application Layer) over TCP/IP.

TCP over UDP

Packet order is critical for drawing coordinates. We chose TCP to guarantee reliable, ordered delivery of the byte stream, preventing corrupted drawings.

Socket Programming

ServerSocket binds to port 5000. Socket initiates the 3-way handshake. Input/Output streams are wrapped in Buffers for string communication.
server = new ServerSocket(5000);
client = new Socket("192.168.x.x", 5000);

Multi-threaded Server

Implemented a Thread-per-Client model. The main loop accepts connections and immediately spawns a ClientHandler thread, preventing blocking.

Custom Protocol

Text-based Application Layer protocol:
JOIN <username>
DRAW <x> <y> <size> <color>
GUESS <word>

0.0.0.0 Binding

Server binds to all network interfaces (0.0.0.0), allowing remote clients on the same Local Area Network (LAN) to connect seamlessly.

Thread Safety

Uses ConcurrentHashMap for the client registry to prevent race conditions during high-concurrency broadcasting.
